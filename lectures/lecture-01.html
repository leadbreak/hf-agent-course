AI 에이전트의 여정

유리병 속 텍스트 생성기를 넘어, 세상을 향해 손을 뻗고 행동하는 인공지능의 진화에 대한 이야기

프롤로그: 알프레드의 탄생 (What is an Agent?)

단순히 질문에 대답만 하던 AI가 어떻게 우리의 물리적, 디지털적 명령을 수행하게 되었을까요? 그 해답은 바로 '에이전트(Agent)'에 있습니다. 에이전트는 사용자가 정의한 목표를 달성하기 위해 환경과 상호작용하는 시스템입니다 [1].

"알프레드, 커피 한 잔 부탁해."
알프레드는 사용자의 자연어를 이해하고(이해), 커피 머신이 있는 주방으로 가야 한다고 계획을 세운 뒤(계획), 실제로 도구를 사용해 커피를 내립니다(행동) [2, 3].

이처럼 에이전트는 추론과 계획을 담당하는 '두뇌(AI 모델)'와 실제 작업을 수행하는 '신체(도구)'의 결합으로 이루어집니다 [1, 4].

▲ 에이전시의 스펙트럼: 단순 정보 처리부터 다중 에이전트까지 [4]

제1장: 에이전트의 두뇌 (LLMs & Messages)

모든 에이전트의 중심에는 대형 언어 모델(LLM)이라는 거대한 두뇌가 자리 잡고 있습니다 [5, 6]. GPT-4나 Llama 같은 모델들은 트랜스포머(Transformer) 아키텍처를 기반으로 하며, 이전 토큰들의 시퀀스를 바탕으로 다음 토큰을 예측하도록 훈련되었습니다 [7-9].

하지만 두뇌가 올바른 방향으로 생각하게 만들려면 명확한 지침이 필요합니다. 우리는 시스템 메시지(System Prompt)를 통해 에이전트에게 성격과 행동 지침을 부여합니다 [10, 11]. 이 시스템 메시지 안에는 에이전트가 사용할 수 있는 도구 목록과 생각하는 방식에 대한 규칙이 포함됩니다 [11, 12].

제2장: 유리병을 깨는 도구 (Tools)

LLM은 매우 똑똑하지만 본질적으로 텍스트만 생성할 수 있다는 한계를 가집니다 [13, 14]. 인터넷에 접속해 최신 날씨를 알아내거나 복잡한 수학 계산을 하려면 '도구(Tools)'라는 신체가 필요합니다.

도구는 웹 검색, API 인터페이스, 계산기 등 LLM에게 부여된 명확한 목적의 함수입니다 [15, 16]. 우리는 LLM에게 도구의 이름, 설명, 예상되는 입력값 및 출력값의 형태를 텍스트로 자세히 설명하여 줍니다 [17, 18]. 그러면 LLM은 직접 행동하는 대신, "이 도구를 호출해줘(call weather_tool('Paris'))"라는 텍스트 명령어(Action)를 생성하게 됩니다 [19, 20].

제3장: 끊임없는 순환 (Thought-Action-Observation)

에이전트는 단순히 묻고 답하는 것에 그치지 않고, 복잡한 문제를 해결하기 위해 ReAct (Reasoning + Acting) 접근 방식을 활용하여 내부적인 순환 고리를 형성합니다 [21-23].

1. 사고 (Thought)

내부적인 추론을 통해 문제를 단계별로 분해하고 다음 행동을 결정합니다 [21, 24].

2. 행동 (Action)

결정된 행동을 실행하기 위해 JSON이나 코드를 생성한 뒤, 더 이상의 텍스트 생성을 멈춥니다(Stop and Parse 방식) [25, 26].

3. 관찰 (Observe)

도구의 실행 결과나 에러 메시지 등 환경으로부터 피드백을 받아 에이전트의 기억(문맥)에 업데이트합니다 [27, 28].

이러한 사고 ➔ 행동 ➔ 관찰의 주기는 에이전트가 목표를 완수하거나 최종 답변(Final Answer)을 도출할 때까지 프로그래밍의 `while` 루프처럼 계속 반복됩니다 [23, 29, 30].

제4장: 우리만의 에이전트 구축 (smolagents)

이러한 복잡한 구조를 밑바닥부터 짜는 것은 매우 피곤한 일입니다 [31]. 그래서 우리는 smolagents와 같은 프레임워크를 사용하여 개발을 단순화합니다 [32].

특히 최근에는 단순한 JSON 출력을 넘어, Python 같은 고급 언어로 실행 가능한 코드 블록을 작성하여 행동하는 '코드 에이전트(Code Agent)' 방식이 부상하고 있습니다 [33, 34]. 코드 에이전트는 루프, 조건문 등 복잡한 논리를 표현할 수 있어 JSON 방식보다 훨씬 뛰어난 유연성과 모듈화를 제공합니다 [34].

가장 훌륭한 학습 방법은 직접 만들어보는 것입니다. 제공된 도구를 활용하거나 창의적인 새로운 도구를 만들어 여러분만의 첫 번째 에이전트를 가동해 보세요! [35, 36]
