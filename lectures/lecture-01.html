1. 에이전트의 해부학 (Anatomy)

기존의 LLM은 연산 규칙을 암기한 훌륭한 '두뇌(Brain)'지만, 외부 세계 시스템에 직접 접근할 권한이나 환경(신체)이 없었습니다.

에이전트는 이 두뇌에 Python 샌드박스, 웹 검색기, 데이터베이스 쿼리 실행기와 같은 '도구(Tools)'를 연결한 아키텍처입니다. 이를 통해 단순한 텍스트 생성을 넘어, 실제 시스템 상태를 변경하는 영향력(Agency)을 확보합니다.

3. 최적화의 끝: Code Agent

과거에는 LLM이 도구를 사용할 때 JSON 형태로 매개변수를 출력하도록 강제했습니다. 하지만 JSON은 조건문(If)이나 반복문(For) 등 프로그래밍 로직을 담을 수 없습니다.

최신 아키텍처인 Code Agent는 LLM이 직접 네이티브 Python 코드를 생성하고 실행합니다. 불필요한 파싱 오버헤드를 줄이고, 복잡한 데이터 연산 로직을 단일 루프 안에서 우아하게 처리합니다.
