에이전트란 무엇인가?

AI 에이전트는 환경을 관찰(Observe)하고, 목표를 달성하기 위해 주어진 도구(Tools)를 사용하여 행동(Act)을 결정하는 시스템입니다.

일반적인 프로그램은 정해진 규칙(If-Then)에 따라 실행되지만, 에이전트는 자율성(Autonomy)을 가집니다. 상황이 변하면 그에 맞춰 스스로 다른 전략을 세울 수 있습니다.

에이전트의 3대 핵심 요소

  • 🧠 두뇌 (Brain): 데이터를 처리하고 계획을 수립하는 대규모 언어 모델(LLM)
  • 🛠 도구 (Tools): 에이전트가 외부 세계와 상호작용할 수 있게 해주는 기능들 (웹 검색, 계산기 등)
  • 🌍 환경 (Environment): 에이전트가 인식하고 행동을 가하는 대상
두뇌의 작동 원리

대규모 언어 모델(LLM)과 메시지 구조

LLM은 단순히 텍스트를 생성하는 것을 넘어, 에이전트의 **추론 엔진** 역할을 합니다. 에이전트는 LLM과 소통할 때 단순한 텍스트가 아닌, 구조화된 메시지 형식(ChatML)을 사용합니다.

System Message

에이전트의 역할, 성격, 행동 규칙을 정의합니다.

{"role": "system", "content": "너는 파이썬 코드를 실행할 수 있는 데이터 분석 AI 에이전트야."}

User Message

사용자가 에이전트에게 내리는 명령이나 질문입니다.

{"role": "user", "content": "현재 테슬라 주가를 바탕으로 그래프를 그려줘."}

Assistant Message

에이전트(LLM)의 응답입니다. 이 응답에는 텍스트뿐만 아니라 도구 호출(Tool Calls) 명령이 포함될 수 있습니다.

{"role": "assistant", "content": "네, 테슬라 주가를 조회하겠습니다.", "tool_calls": [{"name": "get_stock_price", "arguments": {"symbol": "TSLA"}}]}

도구 (Tools) 통합하기

LLM 자체는 외부 세계와 단절되어 있으며, 훈련된 데이터 시점 이후의 정보를 알지 못합니다. 도구(Tools)는 이러한 한계를 극복하게 해줍니다.

Python 함수, API 호출, 웹 검색, 코드 인터프리터 등 어떤 기능이든 도구가 될 수 있습니다. 중요한 것은 LLM이 언제 어떤 도구를 써야 할지 알 수 있도록 명확한 설명(Docstrings)을 제공하는 것입니다.

Python으로 도구 정의하기


from smolagents import tool

@tool
def get_weather(location: str) -> str:
    """
    특정 지역의 현재 날씨를 반환합니다.
    Args:
        location: 날씨를 조회할 도시 이름 (예: 'Seoul', 'Paris')
    """
    # API 호출 로직...
    return f"{location}의 날씨는 맑음입니다."

👉 위 코드의 Docstring과 Type Hint는 시스템 프롬프트로 변환되어 LLM에게 전달됩니다.

smolagents로 첫 에이전트 만들기

Hugging Face의 smolagents 라이브러리는 최소한의 코드로 강력한 에이전트를 구축할 수 있게 해줍니다. 간단한 Alfred 에이전트를 만들어 봅시다.

agent.py Python 

from smolagents import CodeAgent, HfApiModel, DuckDuckGoSearchTool

# 1. 사용할 모델 정의 (Hugging Face의 모델 사용)
model = HfApiModel("Qwen/Qwen2.5-Coder-32B-Instruct")

# 2. 에이전트가 사용할 도구 목록
tools = [DuckDuckGoSearchTool()]

# 3. 에이전트 인스턴스화 (CodeAgent는 파이썬 코드를 작성하고 실행하며 문제를 해결합니다)
agent = CodeAgent(tools=tools, model=model)

# 4. 에이전트 실행!
agent.run("최근 AI 에이전트 트렌드에 대해 검색하고 한글로 3줄 요약해줘.")

Concept 01

에이전트(Agent)란 정확히 무엇인가요?

AI 에이전트는 사용자의 프롬프트에 단순한 텍스트로 답하는 것을 넘어, 주어진 목표를 달성하기 위해 자율적으로 계획을 세우고 행동하는 시스템입니다. 일반적인 코딩 로직과 에이전트의 결정적인 차이는 바로 스스로 판단(Reasoning)하는 능력에 있습니다.

에이전트를 구성하는 3가지 핵심 축

  • 🧠

    1. 두뇌 (The Brain / LLM)

    대규모 언어 모델(LLM)이 에이전트의 두뇌 역할을 합니다. 사용자 입력을 이해하고, 상황을 추론하며, 다음 행동(Action)을 결정하는 핵심 엔진입니다.

  • 🛠️

    2. 도구 (Tools)

    LLM 자체는 외부 세계와 단절되어 있습니다. 도구(웹 검색, 파이썬 인터프리터, 날씨 API 등)는 에이전트에게 눈과 손을 달아주어 외부 환경과 상호작용하게 합니다.

  • 🌍

    3. 환경 (Environment)

    에이전트가 코드를 실행하거나 검색 결과를 받아오는 대상 공간입니다. 에이전트는 환경으로부터 관찰(Observation)을 얻어냅니다.

전통적 프로그램 vs AI 에이전트 특성 비교

💡 핵심 인사이트

If-Then 규칙으로 하드코딩된 프로그램은 예측 가능성이 100%이지만, 예상치 못한 엣지 케이스에서 쉽게 실패합니다. 반면 에이전트는 목표를 주면 경로를 스스로 탐색하기 때문에 훨씬 더 유연하고 복잡한 문제 해결에 적합합니다.

Concept 02

LLM의 역할과 특수 토큰(Special Tokens)

에이전트의 두뇌인 LLM은 인간의 언어뿐만 아니라 코드를 작성하고 도구를 호출하는 데 탁월한 능력을 가지고 있습니다. 이를 위해 LLM은 단순 텍스트가 아닌 구조화된 메시지(ChatML 등) 형태로 소통합니다.

⚙️

System Message

에이전트의 정체성, 행동 지침, 그리고 사용 가능한 도구들의 명세(도구 이름, 파라미터 설명 등)를 제공하는 가장 중요한 백그라운드 지시 사항입니다.

{
  "role": "system",
  "content": "You are a helpful assistant with access to the following tools: \n1. get_weather(location)"
}
👤

User Message

우리가 에이전트에게 내리는 명령이나 질문입니다. 모델은 이 메시지와 이전 대화 내역(Context)을 바탕으로 다음에 해야 할 일을 추론합니다.

{
  "role": "user",
  "content": "서울의 현재 날씨는 어때?"
}
🤖

Assistant Message

LLM이 생성한 응답입니다. 단순한 텍스트 답변일 수도 있지만, "이 도구를 이렇게 실행해라"라는 특수한 도구 호출 명령(Tool Calls)을 포함할 수 있습니다.

{
  "role": "assistant",
  "tool_calls": [{
    "name": "get_weather",
    "arguments": {"location": "Seoul"}
  }]
}

💡 특수 토큰과 ChatML 포맷

LLM은 내부적으로 이 JSON 메시지들을 연속된 문자열로 변환합니다. 이때 시스템 프롬프트의 시작과 끝, 사용자 메시지의 시작과 끝을 구분하기 위해 <|im_start|>, <|im_end|> 같은 특수한 형태의 제어 토큰(Special Tokens)을 사용합니다. 이를 통해 모델은 대화의 맥락을 완벽하게 파싱합니다.

Concept 03

도구 (Tools): 에이전트의 손과 발

훌륭한 목수에게 좋은 연장이 필요하듯, 강력한 에이전트를 만들기 위해서는 명확하게 정의된 도구를 제공해야 합니다. 파이썬 함수부터 외부 API까지 에이전트가 실행할 수 있는 모든 기능이 도구가 됩니다.

python_tool.py Python 
from smolagents import tool

@tool
def calculate_trip_budget(destination: str, days: int) -> int:
    """
    특정 목적지로의 여행 예산을 계산하는 도구입니다.
    
    Args:
        destination: 여행 목적지 (예: 'Tokyo', 'New York')
        days: 여행 일정 (일 단위)
        
    Returns:
        int: 예상되는 총 예산 (달러)
    """
    # 실제 환경에서는 실시간 API를 호출하여 계산할 것입니다.
    base_cost = 500
    daily_cost = 150 if destination == 'New York' else 100
    
    return base_cost + (daily_cost * days)

가장 중요한 것은 '명확한 설명(Docstring)'

코드를 작성하는 우리 눈에는 calculate_trip_budget이라는 이름이 직관적일 수 있습니다. 하지만 LLM에게는 이 함수가 정확히 무엇을 하는지, 인자(Arguments) 타입은 무엇인지, 무엇을 반환하는지 상세히 알려주어야 합니다.

  • 📝

    Docstring의 변환

    에이전트 프레임워크(smolagents 등)는 파이썬 함수의 Docstring과 Type Hints를 파싱하여, 앞서 살펴본 System Message에 도구 설명서로 자동 삽입합니다.

  • ⚠️

    나쁜 설명 = 멍청한 에이전트

    설명이 부족하거나 모호하면, LLM은 엉뚱한 값을 인자로 넘기거나(Hallucination), 도구를 사용해야 할 타이밍을 놓치게 됩니다.

Hands-On Tutorial

smolagents로 첫 에이전트 출격시키기

모든 이론을 배웠으니, 이제 Hugging Face가 만든 가볍고 강력한 라이브러리인 smolagents를 이용해 단 10줄의 코드로 CodeAgent를 만들어 보겠습니다. CodeAgent는 직접 파이썬 코드를 작성하고 실행하며 문제를 해결하는 최신 트렌드의 에이전트입니다.

my_first_agent.py Python 
# 1. smolagents 모듈 임포트
from smolagents import CodeAgent, HfApiModel, DuckDuckGoSearchTool

# 2. 두뇌 역할의 LLM 선택 (Hugging Face 서버리스 API 활용, 무료!)
# Qwen2.5-Coder는 코딩 및 에이전트 태스크에 매우 뛰어난 오픈소스 모델입니다.
model = HfApiModel("Qwen/Qwen2.5-Coder-32B-Instruct")

# 3. 에이전트가 사용할 도구 생성 (여기서는 웹 검색 도구)
tools = [DuckDuckGoSearchTool()]

# 4. 에이전트 초기화! 
# CodeAgent는 생각한 로직을 파이썬 코드로 작성한 후, 샌드박스에서 실행하여 결과를 관찰합니다.
agent = CodeAgent(tools=tools, model=model)

# 5. 미션 부여 및 실행
result = agent.run("프랑스 파리의 현재 날씨는 어때? 검색 도구를 사용해서 알려줘.")

print(f"에이전트의 최종 답변: {result}")

에이전트(Agent)란 정확히 무엇인가요?

AI 에이전트는 사용자의 프롬프트에 단순한 텍스트로 답하는 것을 넘어, 주어진 목표를 달성하기 위해 자율적으로 계획을 세우고 행동하는 시스템입니다. 일반적인 코딩 로직과 에이전트의 결정적인 차이는 바로 스스로 판단(Reasoning)하는 능력에 있습니다.

에이전트를 구성하는 3가지 핵심 축

  • 🧠

    1. 두뇌 (The Brain / LLM)

    대규모 언어 모델(LLM)이 에이전트의 두뇌 역할을 합니다. 상황을 추론하며 행동을 결정합니다.

  • 🛠️

    2. 도구 (Tools)

    도구는 에이전트에게 눈과 손을 달아주어 외부 환경과 상호작용하게 합니다.

  • 🌍

    3. 환경 (Environment)

    에이전트가 코드를 실행하거나 검색 결과를 받아오는 대상 공간입니다.

👤

User Message

에이전트에게 내리는 명령.

🤖

Assistant Message

LLM의 응답. 도구 호출 포함 가능.

{"role": "assistant", "tool_calls": [...]}

도구 (Tools): 에이전트의 손과 발

python_tool.py Python 
from smolagents import tool

@tool
def calculate_trip(dest: str, days: int) -> int:
    """여행 예산을 계산합니다."""
    return 500 + (100 * days)

가장 중요한 것은 '명확한 설명(Docstring)'

Docstring이 System Message에 포함되어 LLM이 도구를 언제/어떻게 쓸지 판단하게 돕습니다.

smolagents로 첫 에이전트 출격시키기

agent.py 
from smolagents import CodeAgent, HfApiModel, DuckDuckGoSearchTool
model = HfApiModel("Qwen/Qwen2.5-Coder-32B-Instruct")
tools = [DuckDuckGoSearchTool()]
agent = CodeAgent(tools=tools, model=model)
result = agent.run("프랑스 파리의 현재 날씨는?")
print(result)
