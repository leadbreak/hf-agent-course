LLM에서 Agent로,
시스템 아키텍처의 전환

정적 언어 모델은 본질적으로 '다음 토큰 예측기'에 불과합니다. 이 추론 엔진이 어떻게 실행 가능한 코드를 매개로 디지털 환경을 직접 제어하는 자율형 에이전트로 거듭나는지, 그 기저의 메커니즘을 파헤칩니다.

1. 에이전트의 해부학 (Anatomy)

대형 언어 모델(LLM)은 방대한 데이터로 학습된 훌륭한 '두뇌(Brain)'입니다. 그러나 이 두뇌는 치명적인 약점을 가집니다. 학습 시점(Cut-off) 이후의 세계를 알지 못하며, 복잡한 수학 연산에서 환각(Hallucination)을 일으키고, 데이터베이스나 외부 API에 직접 접근할 권한이 없습니다.

에이전트 아키텍처는 이 고립된 두뇌에 '신체(Tools)'를 연결하는 작업입니다. Python 샌드박스, 웹 검색 크롤러, SQL 쿼리 실행기 등 외부 시스템과 상호작용할 수 있는 도구를 제공함으로써, 단순 텍스트 생성을 넘어 현실 세계의 상태를 변경하는 에이전시(Agency)를 시스템에 부여합니다.

2. 무한 실행 엔진: ReAct 사이클과 Stop Sequence

가장 중요한 질문은 이것입니다. "문자열만 뱉어내는 LLM이 도대체 어떻게 파이썬 함수를 '실행'하는가?"

그 비밀은 프롬프트 엔지니어링과 파서(Parser)의 결합에 있습니다. 시스템은 아래와 같은 엄격한 형식의 프롬프트를 LLM에 주입하여 사고(Thought) ➔ 행동(Action) ➔ 관찰(Observation)의 ReAct 패턴을 강제합니다.

# Agent의 System Prompt 예시 You are a helpful assistant with access to the following tools: - get_weather(location: str): Returns weather info. - calculate(expression: str): Evaluates math expressions. Use the following format strictly: Thought: Consider what to do next. Action: The action to take, must be one of [get_weather, calculate] Action Input: The input to the action Observation: The result of the action (This will be provided by the system)

LLM이 텍스트를 스트리밍하며 생성하다가 Action Input: [데이터] 부분까지 출력하면, 시스템은 생성을 강제 중단(Stop Sequence) 시킵니다. 이후 생성된 문자열을 파싱하여 로컬 환경에서 도구를 대신 실행하고, 그 결과값을 Observation: 뒤에 이어붙여 LLM에게 다시 전달합니다. 목표가 달성될 때까지 이 루프는 멈추지 않습니다.

3. 최적화의 끝: Code Agent의 등장

과거 OpenAI의 Function Calling을 비롯한 초기 에이전트 모델들은 도구를 호출할 때 JSON 스키마를 생성하도록 설계되었습니다. 하지만 이 방식은 실무 환경에서 심각한 오버헤드와 한계를 드러냈습니다.

최신 아키텍처인 Code Agent는 LLM이 직접 네이티브 Python 코드를 생성하고 실행하는 패러다임입니다.

  • 단일 루프 처리: JSON으로는 불가능한 `For` 반복문이나 `If` 조건문을 사용하여 여러 도구를 한 번의 호출로 엮을 수 있습니다.
  • 변수 상태 저장: 도구 A의 출력값을 변수에 담아 도구 B의 입력값으로 바로 넘길 수 있어, 컨텍스트 윈도우 낭비를 막습니다.
  • 명확한 디버깅: 파싱 에러가 발생했을 때, 스키마 오류를 분석하는 대신 파이썬 고유의 Traceback 로그를 LLM에게 그대로 피드백할 수 있습니다.

# Code Agent가 생성하는 Action 예시 (Python Code) Thought: I need to fetch the data, filter it by year, and calculate the average. Doing this with JSON would require 3 separate ReAct loops. I will do it all in one Python block. Action: def process_data(): raw_data = fetch_db_records(table="sales_2026") filtered = [x for x in raw_data if x['status'] == 'completed'] avg_value = sum(x['amount'] for x in filtered) / len(filtered) return f"The average completed sales value is {avg_value}" result = process_data()
