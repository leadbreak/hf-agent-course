에이전트의 해부학

전통적인 LLM은 훌륭한 '두뇌(Brain)'입니다. 하지만 팔다리가 없어 외부 세계에 영향을 미치지 못했죠.

에이전트는 이 두뇌에 '신체(Tools)'를 연결한 시스템입니다. 웹을 검색하고, 코드를 실행하고, API를 호출하여 현실의 문제를 직접 해결합니다.

왜 Code Agent 인가?

도구를 호출할 때 LLM이 JSON 객체를 내뱉게 하는 방식은 과거의 유물입니다.

이제는 LLM이 직접 Python 코드를 작성하여 도구를 제어합니다. 루프(For/While)와 조건문(If)을 사용할 수 있어 복잡한 논리 처리에 압도적인 성능을 냅니다.

복잡한 로직 처리 (If/For문) JSON vs Code
에러 발생 시 추적 및 복구력 (Stack Trace)
다중 도구 연속 호출 유연성
